Fixed relative-path bug in _run_smoke (setuploop.py): env_dir was a relative Path (shown as "runs/..." in create_env.log), so the PATH entry "runs/.../env/bin" was resolved by bash relative to the subprocess cwd (repo_dir), not the harness cwd — causing "auto-round: command not found" every smoke run. Fix: call env_dir.resolve() to make the PATH entry absolute before embedding it in the subprocess env.
